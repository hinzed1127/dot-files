Create a pull request of this branch, and use @.github/pull_request_template.md as the starting point for the pull request's description. When you fill out the description:

- Keep things brief and to the point.
- Don't include things that I can already view in the GitHub UI, such as the files changed and the number of lines added and deleted for each changed file
- The most important thing is straightforward instructions for the steps to test that are easy to reproduce.
- If I haven't provided it, make sure you ask for the ticket number, so that can be included in the description (see the section requesting a "Link to ticket in ADO"). The commit messages should also provide this by the Azure DevOps convention of linking the ticket via "[AB#ticketNumber]" in the commit message title. If the commits don't include this tag, e.g "feat[AB#ticketNumber]: add something new", ask me to clarify a ticket number or if it's not applicable for the given commit(s).
